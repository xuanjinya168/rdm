# Windows 冒烟测试

本清单用于发布前验证 RDM 的桌面运行期行为。自动化测试通过不等于本清单通过；
托盘、通知、剪贴板、单实例和安装程序必须在真实 Windows 桌面中检查。

构建并启动本地测试服务器：

```powershell
cd apps/rdm-desktop
npm ci
npm run app:build
npm run smoke:exe
npm run app:bundle
npm run smoke:bundle
npm run smoke:server
```

`smoke:exe` 会在隔离数据目录中驱动真实 release EXE，自动验证首启 URL、Range
下载、无 Range 降级、暂停继续、异常恢复、单实例、校验、删除和设置持久化。
当前共 20 项，并覆盖关闭到托盘及第二实例恢复窗口；运行前需退出其他 RDM 实例。

`smoke:bundle` 会管理解包 MSI，并将 NSIS 静默安装到临时目录后卸载。为避免覆盖
现有安装，检测到已安装的 RDM 时会拒绝运行。

随后启动手工测试服务器；它会打印测试 URL 和每个文件的 SHA-256。保持该窗口运行，在另一个
PowerShell 窗口启动：

```powershell
.\apps\rdm-desktop\src-tauri\target\release\rdm-desktop.exe
```

建议使用一个空目录作为下载目录。每项填写 `PASS`、`FAIL` 或 `BLOCKED`，失败时
记录复现步骤和 `%LOCALAPPDATA%\com.rdm.desktop\logs\rdm.log` 中的相关日志。

## 2. 下载核心

| 结果 | 场景 | 操作与验收 |
|---|---|---|
| ___ | 多连接下载 | 下载 `/range.bin`，确认使用多连接并正常完成 |
| ___ | 暂停与继续 | 下载 `/slow.bin`，进度变化后暂停；等待 5 秒进度不再变化，再继续并完成 |
| ___ | 重启续传 | 下载 `/slow.bin`，中途退出 RDM，重新启动后继续；不得从零开始 |
| ___ | 无 Range 降级 | 下载 `/no-range.bin`，应自动单连接完成 |
| ___ | 空文件 | 下载 `/empty.bin`，生成 0 字节文件并完成 |
| ___ | 重定向 | 下载 `/redirect.bin`，应跟随重定向并完成 |
| ___ | 服务端错误 | 下载 `/error.bin`，重试后进入失败状态且界面可继续操作 |
| ___ | SHA-256 正确 | 使用服务器输出的 `/range.bin` SHA-256，任务应完成 |
| ___ | SHA-256 错误 | 将 SHA-256 最后一位改掉，任务应失败且不得发布最终文件 |
| ___ | 文件重名 | 连续下载两次同名文件，不覆盖已有文件 |
| ___ | 取消任务 | 下载 `/slow.bin` 后取消，状态正确且任务可删除 |
| ___ | 删除记录 | 选择“仅删记录”，已完成文件仍保留 |
| ___ | 删除文件 | 选择“删除文件”，记录和对应文件都被删除 |

默认服务器地址为 `http://127.0.0.1:18080`，例如：

```text
http://127.0.0.1:18080/range.bin
http://127.0.0.1:18080/slow.bin
http://127.0.0.1:18080/no-range.bin
```
